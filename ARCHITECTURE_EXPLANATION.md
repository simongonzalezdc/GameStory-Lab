# Why Projects AND Concepts?

## The Two-Level Structure

### Projects = Game Ideas
A **Project** represents a single game idea or game title. Think of it like:
- A folder for "My Epic RPG"
- A container for "Cozy Farming Sim"
- The top-level organization

### Concepts = Versions/Iterations
A **Concept** is a specific version or iteration of that game design. Think of it like:
- Version 1: Initial concept with basic mechanics
- Version 2: Added lore and refined mechanics
- Version 3: Major redesign after playtesting feedback
- Version 4: Final polished version ready for development

## Why This Structure?

### 1. **Version Control for Game Design**
Just like code has git commits, game concepts need versioning:
- Compare different approaches
- See what changed between iterations
- Roll back to a previous version if needed
- Track the evolution of your design

### 2. **Multiple Approaches**
You might want to try different directions:
- Concept v1: Dark, serious RPG
- Concept v2: Light-hearted, comedic RPG (same project, different take)
- Compare which works better

### 3. **Iterative Refinement**
The refinement process creates new versions:
- Start with v1 (rough)
- Refine → v2 (better)
- Refine again → v3 (polished)
- Each refinement is a new concept version

### 4. **Future Multi-User Support**
When teams are added:
- Multiple people can work on the same project
- Each person can create their own concept versions
- Compare team members' approaches

## Is It Overkill?

**For solo use:** Maybe! You could simplify to just concepts if you want.

**For teams/iterations:** No, it's useful because:
- You naturally create multiple versions when refining
- You want to compare "what if I changed X?"
- You need to track design evolution

## Current Limitations

Right now, the UI doesn't make this super obvious:
- ❌ No easy way to see all concepts at once
- ❌ No version comparison view
- ❌ No concept deletion
- ❌ Hard to understand why you have 4 versions

## Proposed Simplification (Optional)

If you want to simplify for solo use, we could:
1. **Hide projects** - Just show concepts directly
2. **Auto-create project** - When you create a concept, auto-create a project
3. **Single concept mode** - Only show the latest concept, hide versioning

But the current structure is actually useful once you start refining and iterating!
