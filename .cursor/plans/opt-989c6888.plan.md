# Architect Documentation Fetch Plan

## 1. Prevent Duplicate Checks in Strict Mode

- In `packages/frontend/src/pages/ProjectArchitectPage.tsx`, introduce a `useRef` (e.g., `checkedProjects`) to track which projectIds have already triggered `checkDocumentation`.
- Update `checkDocumentation` to accept an optional `{ force?: boolean }` flag. Skip the fetch when `force` is false and the projectId is already marked as checked.
- Call `checkDocumentation()` from `useEffect` (when `projectId` changes) without the force flag so the second Strict Mode invocation is ignored.

## 2. Allow Explicit Refresh After Proposals

- When documentation is updated via `onProposalAccepted`, call `checkDocumentation({ force: true })` and clear/update the tracking ref so a fresh fetch occurs.
- Confirm the behavior: initial mount logs at most once, while manual refreshes still hit the endpoint.

## 3. Verify UX

- Ensure the UI still loads documents when they exist and that no new console errors are introduced. Optionally log (only once) when documentation is missing to aid debugging.
