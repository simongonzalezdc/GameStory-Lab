# Fix Backend Compilation Error

## Problem

The backend server is failing to start with this error:

```
ERROR: The symbol "sanitizedUpdates" has already been declared
at architect-service.ts:217:10
```

This causes all frontend API requests to fail with "Failed to fetch" / "IMPL_CONNECTION_REFUSED" errors since the backend is not running.

## Root Cause

In `packages/backend/src/services/architect/architect-service.ts`, the `applyAssistantUpdates` method has two declarations of `sanitizedUpdates`:

- Line 172: First declaration (correct, used for validation and new package creation)
- Line 217: Duplicate declaration (incorrect, should reuse the variable from line 172)

## Solution

Remove the duplicate declaration on line 217. The variable is already available from line 172 at the method's start, so the second `const sanitizedUpdates =` is unnecessary and causes the compilation error.

Change line 217 from:

```typescript
const sanitizedUpdates = this.normalizeDocumentUpdates(updates);
```

to simply use the existing `sanitizedUpdates` variable (no declaration needed).

## Files to Modify

- `packages/backend/src/services/architect/architect-service.ts` - Remove duplicate declaration on line 217

## Expected Outcome

After fixing the duplicate declaration:

- Backend server will compile successfully
- Backend will start on port 3007
- Frontend will be able to connect to the API
- All "Failed to fetch" errors will be resolved
