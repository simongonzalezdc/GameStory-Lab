# 🎮 GameStory Lab - User Tutorial

Welcome to GameStory Lab! This tutorial will walk you through creating your first game concept from start to finish.

## What You'll Learn

By the end of this tutorial, you'll know how to:
- Create a project
- Generate game concepts with AI
- Understand mechanics vs lore
- Validate your concept for consistency
- Refine and iterate on your design
- Export your concept to professional documentation

---

## Part 1: Understanding the Basics

### What is a Project?

A **Project** is a container for your game ideas. Think of it as a folder where you keep all your concepts for a single game.

### What is a Concept?

A **Concept** is a complete game design that includes:
- **Mechanics**: How the game plays (core loop, player actions, progression systems, win/lose conditions)
- **Lore**: The story and world (setting, protagonist, conflict, themes, world rules)

Concepts are **versioned**, meaning you can create multiple versions and refine them over time.

### The Workflow

1. **Create a Project** → Give your game a name and genre
2. **Generate a Concept** → Use AI to create mechanics and/or lore
3. **Validate** → Check for consistency issues
4. **Refine** → Improve your concept with AI assistance
5. **Export** → Generate professional documentation (GDD, pitch deck, etc.)

## Part 2: Creating Your First Project

### Step 1: Navigate to Projects

When you first open GameStory Lab, you'll see the **Projects** page. This is your dashboard where all your game projects are listed.

### Step 2: Create a New Project

1. Click the **"+ Create New Project"** button (top of the page)
2. Enter a **Project Title** (required)
   - Example: "My Epic RPG" or "Cozy Farming Sim"
3. Optionally add a **Genre or Mood**
   - Example: "RPG", "cozy sim", "tactical strategy"
   - This helps AI generate more appropriate content
4. Click **"Create Project"**

You'll now see your project in the list!

---

## Part 3: Generating Your First Concept

### Option A: Generate from Scratch (Recommended for First Time)

1. Click on your project to open it
2. You'll see an empty state with two options:
   - **"Generate Concept with AI"** ← Click this!
   - **"Start from Template"** (we'll cover this later)

3. In the generation modal:
   - **What to generate?**
     - **Both** (recommended): Creates both mechanics and lore together
     - **Mechanics only**: Just gameplay systems
     - **Lore only**: Just story and world
   
   - **Describe your game idea** (optional but helpful):
     - Example: "A cozy farming sim where players manage a small farm, build relationships with villagers, and explore a magical forest"
     - The more detail you provide, the better the AI can tailor the concept

4. Click **"Generate Concept"**
5. Wait a few moments while AI creates your concept...

### Option B: Start from a Template

1. Click **"Templates"** in the navigation
2. Browse the available genre templates:
   - **RPG**: Fantasy role-playing game template
   - **FPS**: First-person shooter template
   - **Strategy**: Tactical strategy game template
   - **Puzzle**: Puzzle game template
   - **Survival**: Survival game template

3. Click on a template to preview it
4. Click **"Create Project from Template"**
5. Enter a project name
6. The template will be customized and added as your first concept

## Part 4: Understanding Your Concept

Once your concept is generated, you'll see three tabs:

### Mechanics Tab

Shows the **gameplay systems**:
- **Core Loop**: The main gameplay cycle
- **Player Actions**: What players can do
- **Progression Systems**: How players advance
- **Win/Lose Conditions**: How players succeed or fail
- **Resource Systems**: Items, currency, materials, etc.

### Lore Tab

Shows the **story and world**:
- **Setting**: Era, location, world type
- **Protagonist**: Background, motivation, abilities
- **Conflict**: Primary and secondary challenges
- **World Rules**: Physics, magic, technology
- **Themes**: Central narrative themes

### Validation Tab

Shows **consistency checks**:
- **Consistency Score**: Overall quality (0-100%)
- **Issues**: Any problems found (errors, warnings, info)
- **Suggestions**: How to fix issues

---

## Part 5: Validating Your Concept

### Understanding Validation

The validation system checks if your mechanics and lore work together coherently. It looks for:
- **Mechanics-Lore Alignment**: Do the gameplay actions match the story?
- **Genre Conventions**: Does it follow genre expectations?
- **World Physics**: Are the world rules consistent?
- **Progression Coherence**: Does progression make sense?
- **Narrative Structure**: Is the story well-structured?
- **Technical Feasibility**: Can this actually be built?

### Checking Your Score

1. Click the **"Validation"** tab
2. Look at your **Consistency Score**:
   - **80-100%**: Excellent! Your concept is very coherent
   - **60-79%**: Good, but some improvements possible
   - **Below 60%**: Needs work - check the issues below

3. Review any **issues** found:
   - **Errors** (red): Critical problems that should be fixed
   - **Warnings** (yellow): Potential issues to consider
   - **Info** (blue): Suggestions for improvement

### Fixing Issues

You can fix issues by:
1. **Refining** your concept (see Part 6)
2. **Manually editing** (if you have direct editing access)
3. **Regenerating** specific parts

---

## Part 6: Refining Your Concept

### What is Refinement?

Refinement uses AI to improve your concept while keeping what works. It creates a **new version** so you can compare changes.

### How to Refine

1. Click the **"✨ Refine"** button (top right)
2. Choose a focus:
   - **Deepen Mechanics**: Add more detail to gameplay systems
   - **Enrich Lore**: Expand the story and world
   - **Improve Consistency**: Fix validation issues
   - **Enhance Genre Fit**: Better align with genre conventions

3. Wait while AI refines your concept...
4. A new version is created automatically
5. Compare versions to see what changed

### Version History

- Each refinement creates a new **version**
- You can see all versions in the project
- The latest version is shown by default
- Versions are numbered (v1, v2, v3, etc.)

---

## Part 7: Exporting Your Concept

### Export Formats

Once you're happy with your concept, export it to professional documentation:

1. Click the **"📄 Export"** button (top right)
2. Choose a template:
   - **GDD** (Game Design Document): Comprehensive design doc
   - **Pitch**: Investor/publisher pitch deck
   - **Technical**: Technical specification document

3. The markdown file downloads automatically
4. Open it in any markdown viewer or editor

### What's Included

The exported document includes:
- Project overview
- Complete mechanics breakdown
- Full lore and narrative
- Validation results
- Consistency score
- All concept details in a professional format

## Part 8: Advanced Features

### Working with Multiple Concepts

- A project can have **multiple concepts** (versions)
- Each concept is independent
- Use refinement to create new versions
- Compare different approaches

### Using Templates

Templates are pre-built starting points:
- **Browse Templates**: See all available genres
- **Preview**: Check what's included before creating
- **Customize**: Templates can be modified after creation

### AI Model Selection

The system automatically selects the best AI model:
- **Ollama** (free, local): Used when available
- **OpenRouter** (cloud): Access to multiple models
- **Google Gemini**: Fast validation

You can't manually select models yet, but the system chooses optimally.

## Part 9: Tips & Best Practices

### Getting Better Results

1. **Be Specific**: The more detail in your prompt, the better
2. **Use Genre**: Always set a genre for better AI guidance
3. **Iterate**: Don't expect perfection on the first try
4. **Validate Early**: Check consistency before refining too much
5. **Refine Strategically**: Use the right refinement focus for your needs

### Common Workflows

**Workflow 1: Start Fresh**
1. Create project
2. Generate "Both" (mechanics + lore)
3. Validate
4. Refine based on issues
5. Export when satisfied

**Workflow 2: Mechanics First**
1. Create project
2. Generate "Mechanics only"
3. Review gameplay
4. Generate "Lore" (uses mechanics as context)
5. Validate and refine

**Workflow 3: Template-Based**
1. Browse templates
2. Create project from template
3. Customize and refine
4. Export

### Troubleshooting

**Low Consistency Score?**
- Check validation issues
- Use "Improve Consistency" refinement
- Make sure mechanics and lore align

**AI Generation Failed?**
- Check System Status page
- Ensure Ollama is running (if using local AI)
- Check API keys (if using cloud AI)

**Can't See Concepts?**
- Make sure you generated a concept
- Check that the project loaded correctly
- Try refreshing the page
